# DM Claude — Расширенный форк

> **Форк [Sstobo/Claude-Code-Game-Master](https://github.com/Sstobo/Claude-Code-Game-Master)** с модульной архитектурой — включай опциональные системы (выживание, огнестрел, встречи, навигация) для каждой кампании как моды.

**Закинь любую книгу. Играй внутри истории.** Любимый фэнтези-роман? Фанфик по STALKER? Странная научная фантастика 70-х? Закидываешь PDF, и DM Claude извлекает всех персонажей, локации, предметы и сюжетные линии, а потом бросает тебя в этот мир за кого захочешь.

Правила D&D 5e придают истории ставки и последствия. Знать D&D не нужно — просто говори, что хочешь сделать.

## Модули

Опциональные системы, включаемые для каждой кампании. При создании `/dm` сканирует доступные модули и показывает меню выбора — выбирай что подходит под сеттинг, DM сам пропишет настройки.

| Модуль | Что делает | Подходит для |
|--------|-----------|--------------|
| 🌍 **world-travel** | Пространственная симуляция мира: реальные XY-координаты, A* поиск пути, время пути по расстоянию + скорости. Автоматически запускает проверку встреч при каждом перемещении. ASCII-карта, мини-карта, GUI. | Любая кампания с картой и путешествиями |
| 🎭 **custom-stats** | Любые кастомные статы — мана, рассудок, кислород, репутация, голод, радиация. Убывание/прирост в час, условные эффекты (артефакт лечит только пока HP < max), посекундная симуляция. Ничего не захардкожено. | STALKER, Fallout, хоррор, фэнтези |
| ⚔️ **firearms-combat** | Автоматизированный резолвер боя. RPM → выстрелов за раунд, огневые режимы (одиночный/очередь/авто), PEN vs PROT, бонусы подкласса. Шаблон с АКМ/АК-74/M4A1/СВД. | Любая современная/военная кампания |
| 📦 **inventory-system** | Атомарные транзакции (`--gold --hp --xp --add --remove` одной командой). Stackable предметы с количеством. Unique предметы (оружие, броня). Режим `--test`. | Все кампании (рекомендуется всегда) |

Каждый модуль самодостаточен: свои `tools/`, `lib/`, `rules.md`, `module.json`. Брось папку в `.claude/additional/modules/` — и модуль установлен.

## Что нового в этом форке

### 🧩 Модульная система — Моды для кампаний
При создании новой кампании `/dm` сканирует доступные модули и предлагает их как опциональные моды. Выбираешь нужное — DM включает их и автоматически патчит конфиг кампании.

```
================================================================
  ОПЦИОНАЛЬНЫЕ МОДЫ
  ────────────────────────────────────────────────────────────
  [1] World Travel         — карта, расстояния, время в пути + встречи
  [2] Custom Stats         — любые статы (голод, рассудок, мана) в час
  [3] Firearms Combat      — современное оружие, режимы огня, PEN/PROT
  [4] Inventory System     — атомарные транзакции, stackable/unique предметы
  [A] Включить ВСЕ
  [N] Ничего — стандартный D&D
================================================================
```

Каждый модуль самодостаточен: свои инструменты, правила и патчи конфига. Добавь community-модуль, бросив папку в `.claude/additional/modules/`.

### 🔀 Middleware архитектура
CORE инструменты (`dm-time.sh`, `dm-player.sh`, `dm-session.sh` и т.д.) — vanilla upstream. Модули подключаются через middleware — без изменения CORE. `/dm` автоматически инжектирует правила всех активных модулей в контекст.

### 🌍 Модуль World Travel
`coordinate-navigation` и `encounter-system` объединены в один модуль `world-travel`. Один модуль — всё в комплекте: пространственные координаты, A* поиск пути, расчёт времени пути и автоматические проверки встреч при каждом перемещении.

```bash
bash tools/dm-session.sh move "Руины"
# Авто: считает расстояние → тикает время → проверяет встречу → применяет эффекты статов
```

### ⏱️ Отслеживание времени и отложенные последствия
`dm-time.sh` теперь хранит суммарное прошедшее время кампании. Последствия можно привязать к количеству часов — `tick` уменьшает счётчик при каждом продвижении времени, срабатывает при достижении нуля.

```bash
bash tools/dm-time.sh advance 4 --elapsed
# Хранит total_hours_elapsed, автоматически тикает все отложенные последствия

bash tools/dm-consequence.sh add "Торговец прибывает" "24ч" --hours 24
bash tools/dm-consequence.sh tick 4
# Показывает: "Торговец прибывает (20.0ч)" → ... → "СРОЧНО!"
```

### 📜 Создание сюжетов в CORE
`add_plot()` перенесён из модуля `quest-system` в ванильный CORE. Никакой модуль не нужен — `dm-plot.sh add` работает в любой кампании.

```bash
bash tools/dm-plot.sh add "Потерянная реликвия" --type side --description "..."
```

### 🎯 Единый менеджер инвентаря
Атомарная система транзакций для состояния персонажа — применяй несколько изменений разом или ни одного. Stackable предметы (расходники с количеством) и unique предметы (оружие, броня, квестовые предметы) с автоматической валидацией и откатом.

```bash
# Атомарная транзакция — всё или ничего
bash .claude/additional/modules/inventory-system/tools/dm-inventory.sh update "Персонаж" \
  --gold +150 \
  --xp +200 \
  --hp -10 \
  --add "Аптечка" 2 "Патроны 9мм" 30 \
  --add-unique "АК-47 (7.62x39мм, 2d8+3, PEN 4)" \
  --remove "Бинт" 1 \
  --custom-stat hunger +20

# Тест-режим — проверка без применения
bash .claude/additional/modules/inventory-system/tools/dm-inventory.sh update "Персонаж" --gold -500 --test
```

**Автоматическая миграция** из старого формата инвентаря (создаёт backup с timestamp).

### ⚔️ Автоматизированный огневой бой
Разрешение боя с огнестрельным оружием с реалистичными механиками — расчёт выстрелов за раунд на основе RPM, огневые режимы (одиночный/очередь/авто), прогрессивные штрафы к атаке, механика PEN vs PROT. Детальный вывод каждого выстрела с полным разбором бросков.

```bash
bash .claude/additional/modules/firearms-combat/tools/dm-combat.sh resolve \
  --attacker "Сталкер" \
  --weapon "АКМ" \
  --fire-mode "full_auto" \
  --ammo 60 \
  --targets "Мутант#1:AC14:HP30:PROT2" "Мутант#2:AC14:HP30:PROT2"

# Тест-режим — предпросмотр боя без изменения состояния
bash .claude/additional/modules/firearms-combat/tools/dm-combat.sh resolve ... --test
```

**Автосохранение** расхода патронов и начисления XP. Учитывает классовые бонусы (например, Стрелок с уменьшенными штрафами).

### 📦 Шаблон современной кампании с огнестрелом
Готовый шаблон (`.claude/additional/modules/firearms-combat/templates/modern-firearms-campaign.json`) с оружием (АКМ, АК-74, M4A1, СВД), типами брони с PROT, определениями огневых режимов, кастомными статами выживания (голод/жажда/радиация/сон), эффектами времени и системой встреч — готово для кампаний в стиле STALKER, Fallout или Cyberpunk.

### Кастомные характеристики персонажа
Определяй **любые** статы для своей кампании — голод, жажда, радиация, мораль, рассудок, репутация — что угодно. Полностью универсально, ничего не захардкожено.

```bash
bash tools/dm-player.sh custom-stat hunger +15
bash tools/dm-player.sh custom-stat radiation -5
```

### Движок эффектов времени
Статы автоматически меняются с течением игрового времени. Задай скорость изменения в час в конфиге кампании — система всё сделает сама. Поддержка **условных эффектов** (например, артефакт лечит только пока HP < max) с посекундной симуляцией.

```
Time updated to: Evening (18:30), Day 3
Custom Stats:
  hunger: 80 → 68 (-12)
  thirst: 70 → 52 (-18)
⚠️ TRIGGERED: "Торговец прибыл"
```

### Автоматическое время перемещения
Перемещаешься между локациями — время пути считается автоматически по расстоянию и скорости персонажа. Статы тикают во время путешествия.

```bash
bash tools/dm-session.sh move "Руины"
# Авто-расчёт: 2000м при 4 км/ч = 30 минут
# Авто-применение эффектов времени к голоду, жажде и т.д.
```

### Отложенные события по времени
Планируй события, которые сработают через X часов игрового времени, а не только по сюжету. Отображение оставшегося времени в минутах/часах/днях.

```bash
bash tools/dm-consequence.sh add "Торговец прибывает" "через 24 часа" --hours 24
# Показывает: "Торговец прибывает (12.5ч осталось)" или "СРОЧНО!"
```

### World Travel — Встречи + навигация в одном
Настраиваемые случайные встречи при перемещении — частота зависит от расстояния, времени суток и характеристик персонажа. Встречи создают путевые точки на карте. У локаций настоящие координаты, A* поиск пути находит маршруты, мир виден как ASCII-карта или в GUI-окне.

```bash
bash tools/dm-session.sh move "Руины"
# Авто: проверяет встречу, считает время пути, тикает статы

bash .claude/additional/modules/world-travel/tools/dm-encounter.sh check "Деревня" "Руины" 2000 open
bash .claude/additional/modules/world-travel/tools/dm-map.sh              # Полная ASCII-карта
bash .claude/additional/modules/world-travel/tools/dm-map.sh --minimap    # Тактическая мини-карта
bash .claude/additional/modules/world-travel/tools/dm-map.sh --gui        # GUI-окно с цветами рельефа

# Добавить локацию по азимуту и расстоянию
bash tools/dm-location.sh add "Форпост" "Заброшенный форпост" \
  --from "Деревня" --bearing 90 --distance 2500 --terrain лес
```

### Поддержка i18n
Кириллические имена, нестандартные отношения NPC и Unicode-идентификаторы работают из коробки. Создавай кампании на любом языке. Весь JSON сохраняется с `ensure_ascii=False` — никакого экранирования `\uXXXX`.

## Как это работает

При импорте документа система векторизует его через ChromaDB и запускает агентов-экстракторов, которые разбирают книгу на структурированные данные. Во время игры каждая сцена основана на реальных отрывках из исходного материала.

Всё сохраняется. NPC помнят, что ты говорил в прошлой сессии. Последствия срабатывают через дни игрового времени. Локации меняются по мере развития событий. Сохраняй и восстанавливай в любой момент.

Специализированные агенты запускаются на лету — статы монстров, механики заклинаний, таблицы лута, базы экипировки. Игрок видит только историю. Используется [D&D 5e API](https://www.dnd5eapi.co/) для официальных правил, заклинаний, монстров и экипировки.

## Планы

Фичи в разработке:

- **Вложенные под-карты** — локации могут содержать собственные внутренние карты. Зашёл в замок — исследуй этажи. Поднялся на корабль — осмотри палубы. Спустился в пещеру — разветвлённые тоннели. Каждая под-карта связана с родительской картой мира.
- **Многоэтажные подземелья** — вертикальная навигация с лестницами, лифтами, люками между этажами. Каждый этаж — своя под-карта с независимым состоянием комнат.
- **Интерьеры транспорта** — корабли, дирижабли, космические станции как исследуемые под-карты, которые перемещаются по карте мира. Транспорт едет между локациями, пока ты исследуешь его изнутри.
- **Вес и слоты инвентаря** — грузоподъёмность на основе STR, штрафы за перегруз, автоматический стакинг, фильтры по категориям. [См. дизайн в TODO.md](TODO.md)
- **Экспорт карт** — экспорт ASCII-карт в PNG/SVG для шаринга вне терминала.
