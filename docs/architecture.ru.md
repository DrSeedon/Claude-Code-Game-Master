# Архитектура и бизнес-логика DM System

## Что это за продукт

Проект запускает постоянную настольную RPG-кампанию через Claude Code или
Codex. Модель отвечает за решения ведущего и повествование, а локальный движок
отвечает за проверяемое состояние: персонажей, инвентарь, карту, время, квесты,
последствия, боевые параметры и сохранения.

Главный принцип: модель не является базой данных. Сначала механическое изменение
записывается инструментом, потом результат описывается игроку.

## Клиенты: Claude и Codex

Claude использует команды из `.claude/commands/`. Codex воспринимает те же
намерения через `codex-skills/dm/SKILL.md`; нативная регистрация slash-команд
ему не нужна. Оба клиента вызывают одни shell-инструменты, компилируют одни
правила из `.claude/additional/` и работают с одними файлами кампании.

Skill и specialist reference являются инструкциями, которые читаются по
необходимости. Они не являются постоянно работающими процессами и сами по себе
не расходуют ресурсы между ходами. Subagent нужен только для действительно
независимой параллельной задачи.

## Путь одного хода

1. DM skill выбирает командный сценарий и компилирует активные правила в
   `/tmp/dm-rules.md`.
2. Модель определяет нужную механику и вызывает один из `tools/dm-*.sh`.
3. Bash-обёртка проверяет middleware активных модулей.
4. Если модуль вернул `64`, действие ему не подходит и вызывается CORE. `0`
   означает, что модуль полностью обработал действие. Другой код является
   ошибкой и не маскируется запуском CORE.
5. Python-слой меняет WorldGraph или метаданные кампании транзакционно.
6. Только после успешной записи модель рассказывает игроку результат.

## Граница кампании

Каждый процесс получает кампанию через `DM_ACTIVE_CAMPAIGN`. Это позволяет
веб-серверу одновременно вести разные кампании без гонки за глобальный
`active-campaign.txt`. CLI без переменной использует глобально выбранную
кампанию. Имена валидируются, поэтому `../`, абсолютные пути и скрытые каталоги
не могут выйти из `world-state/campaigns/`.

## Хранилища

`world.json` является единственным источником истины для игрового мира. В нём
находятся узлы игрока, NPC, локаций, предметов, существ, фактов, квестов,
последствий, способностей и экономики. Типизированные рёбра задают положение,
связи локаций, владение, отношения и зависимости рецептов.

`campaign-overview.json` хранит только метаданные: часы, календарь, активные
модули, стиль рассказчика, режим управления игроком и отображаемую позицию.
`campaign-rules.md` содержит правила конкретной кампании.

`WorldRepository` блокирует `world.json`, пишет через временный файл и
`os.replace`, ведёт ревизию и отклоняет устаревшую запись. `JsonOperations`
делает такую же атомарную транзакцию для метаданных. Поэтому два хода одной
кампании сериализуются, а разные кампании не блокируют друг друга.

## CORE

CORE всегда доступен и не зависит от жанра:

- броски, атаки, спасброски и урон;
- игрок, HP, XP, деньги, состояния и пользовательские показатели;
- NPC, группа и отношения;
- локации и связи;
- инвентарь, расходование, передача и крафт;
- факты, квесты, цели и отложенные последствия;
- игровой календарь и часы;
- производство, периодические доходы и расходы;
- снапшоты, восстановление и сброс кампании.

`WorldGraph.tick(hours)` является единым временным тактом. Он изменяет
пользовательские показатели, эффекты, производство, экономику, последствия и
случайные экономические события. `TimeManager` отдельно меняет отображаемые
часы и дату тем же количеством времени.

## Сцена и перемещение

`dm-scene.sh` является оркестратором сложного сюжетного перехода: проверяет
ссылки, создаёт или связывает локацию, двигает группу, добавляет время,
разрешает последствия и обновляет цели квестов.

Без `world-travel` используется простое CORE-перемещение. С активным модулем
сцена передаёт движение единому адаптеру модуля:

- обычный маршрут считает расстояние, скорость, время и дорожные встречи;
- составная локация ведёт через точку входа;
- транспорт переключает глобальную карту и внутренние помещения.

Таким образом, `dm-scene` не дублирует алгоритм путешествия. Параметр
`--elapsed` обозначает дополнительное время самой сцены и складывается с
рассчитанным дорожным временем.

## Опциональные модули

Модуль включается отдельно для каждой кампании и может заменить отмеченный
слот правил, перехватить CORE-команду или добавить собственную команду.

- `world-travel` добавляет координаты, маршруты, препятствия, встречи,
  составные локации и транспорт.
- `firearms-combat` заменяет модель стрельбы: темп огня, длительность очереди,
  расход боезапаса, отдачу и режимы огня.
- `mass-combat` добавляет подразделения и массовые сражения.

Модули не должны содержать имена и лор конкретной кампании. Их конфигурация
лежит в `module-data`, а игровые сущности всё равно записываются в WorldGraph.

## Режимы кампании

`interactive` оставляет почти каждое действие игроку. `narrative` разрешает
ведущему выполнять рутинные действия персонажа и продолжать рассказ до
существенного выбора, опасности или границы главы. Режим меняет объём агентности,
но не отключает броски, стоимость способностей и сохранение состояния.

Кинематографические изображения являются отдельным skill. Он загружается
только на крупной визуальной сцене и не участвует в механике хода.

## Создание, сохранение и миграция

CLI и веб используют один `CampaignManager`, одну схему overview и один узел
`player:active`. Старый `character.json` больше не создаётся.

Снапшот содержит `world.json`, `campaign-overview.json` и все JSON-файлы
активных модулей. Сброс сначала создаёт проверенный внешний архив и только
после этого пересоздаёт весь каталог кампании.

`legacy_migration.py` переносит старые плоские файлы в граф идемпотентно.
Повторный запуск пропускает уже существующие узлы и связи.

## Веб-слой

FastAPI даёт список кампаний, мастер создания, статус персонажа и WebSocket
игровой сессии. На одну кампанию существует один `GameSession` и один lock
хода: два клиента видят один поток, но не могут одновременно запустить два
изменяющих мир ответа.

Claude SDK получает `DM_ACTIVE_CAMPAIGN`, поэтому его Bash-команды работают
строго в кампании текущего WebSocket. Сервер разрешён без пароля только на
loopback. Для внешнего bind обязателен `DND_AUTH_PASSWORD`.

## RAG

RAG не является хранилищем игрового состояния. Он индексирует импортированные
книги и помогает найти исходный фрагмент или дополнить сущность контекстом.
Если RAG отключён или индекс отсутствует, CORE-кампания продолжает работать.

## Что устарело

Удалены отдельные менеджеры игрока, инвентаря, NPC, локаций, фактов и сюжетов,
потому что они создавали параллельные источники истины. Также удалены второй
CORE-движок встреч и дубликат `ModuleDataManager`.

Файлы в `extracted/` остаются намеренно: это промежуточный формат импорта, а не
живая база мира. `session-handoff.md` тоже не дублирует механику: это краткая
нарративная память для следующей сессии.

## Куда вносить изменения

| Требуется изменить | Правильное место |
|---|---|
| Хранение сущностей, инвентарь, квесты, последствия | `lib/world_graph.py` и узкая обёртка в `tools/` |
| Блокировки и атомарная запись мира | `lib/world_repository.py` |
| Метаданные и схема новой кампании | `lib/campaign_schema.py`, `lib/campaign_manager.py` |
| Один составной сюжетный переход | `lib/scene_manager.py`, `tools/dm-scene.sh` |
| Общие правила ведения игры | `.claude/additional/dm-slots/` |
| Жанровая механика, которую можно выключить | `.claude/additional/modules/<module>/` |
| Лор, имена, особые правила одной кампании | `campaign-rules.md` и узлы её `world.json` |
| Поведение Codex при `/dm` | `codex-skills/dm/` |
| Поведение Claude slash-команды | `.claude/commands/` |
| Импорт и поиск по книгам | `lib/rag/`, `dm-extract.sh`, `dm-search.sh` |

Новый общий gameplay-контракт сначала добавляется в WorldGraph и CORE tool.
Модуль перехватывает его только тогда, когда действительно заменяет механику.
Это сохраняет один источник истины и не создаёт второй менеджер рядом.

## Оставшиеся ограничения

Транзакция одного JSON-файла надёжна, но изменение `world.json` и
`campaign-overview.json` не является общей транзакцией файловой системы. При
аварийном завершении ровно между этими коммитами отображаемая позиция или часы
могут временно расходиться с графом; session preparation и инструменты
перемещения восстанавливают их из игрового состояния.

Проект рассчитан на запуск из репозитория через `uv`, а не на автономную
установку wheel: правила, shell-инструменты и frontend являются частью
репозитория.
