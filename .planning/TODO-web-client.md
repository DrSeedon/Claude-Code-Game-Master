# Web Client — полноценный игровой интерфейс

## Концепция
Заменить Claude Code CLI для игры. Полноценный web-клиент где можно создавать кампании, настраивать их, играть с сохранением чата между сессиями.

## Экраны

### 1. Лобби (главная)
- Список существующих кампаний (из world-state/campaigns/)
- Карточки: название, персонаж, уровень, количество сессий, последняя локация
- Кнопка "Новая кампания"
- Кнопка "Продолжить" для каждой кампании

### 2. Создание кампании (визард)
Аналог /new-game, пошаговый:
- Название кампании
- Выбор модулей (firearms, mass-combat, world-travel)
- Выбор нарратор-стиля (sarcastic-puns, horror, epic, cinematic)
- Выбор шаблона правил (zombie-apocalypse, survival-zone, civilization и т.д.)
- Кастомные правила (realistic-progression, russian-language)
- Валюта и календарь
- Создание персонажа
- Результат: кампания готова, можно играть

### 3. Игровой чат (основной экран)
- Чат игрок ↔ DM с стримингом ответов
- Tool calls видны (кубики, инвентарь, перемещение)
- Сайдбар: персонаж (HP, XP, Gold, Inventory, Custom Stats)
- **Персистентный чат** — история сохраняется между сессиями (как в Claude Code)
- При открытии кампании — загружается последний чат, можно продолжить
- Кнопка "Новая сессия" для чистого чата

### 4. Управление кампанией
- Обзор мира (NPC, локации, квесты, последствия)
- Дашборд персонажа (полный character sheet)
- Wiki (оружие, броня, зелья, рецепты)
- Карта локаций (если world-travel модуль)

## Архитектура

### Backend (FastAPI)
- `POST /api/campaigns` — создать кампанию
- `GET /api/campaigns` — список кампаний
- `GET /api/campaigns/{name}` — инфо кампании
- `DELETE /api/campaigns/{name}` — удалить
- `POST /api/campaigns/{name}/switch` — переключить активную
- `GET /api/campaigns/{name}/status` — статус персонажа
- `GET /api/campaigns/{name}/chat-history` — история чата
- `WebSocket /ws/game/{name}` — игровой чат для конкретной кампании
- Каждая кампания = отдельная LLM сессия (system prompt + conversation history)

### Frontend (React)
- React Router: /lobby, /new-game, /play/:campaign, /manage/:campaign
- Состояние чата сохраняется в localStorage или на сервере (JSON)
- WebSocket per campaign

### Сохранение чата
- Conversation history сохраняется в `world-state/campaigns/{name}/chat-history.json`
- При открытии кампании — загружается последний чат
- Можно начать новую сессию (чистый чат) или продолжить старый
- System prompt перегенерируется каждый раз (правила + состояние мира актуальные)

## Два AI провайдера
- Claude Code SDK (подписка, без API ключа) — для личного использования
- Anthropic API (API key) — для продакшна/SaaS
- Автовыбор: есть ключ → API, нет → SDK

## Что НЕ делать (пока)
- Auth/billing/multi-user (это SaaS, отдельный проект)
- Deploy в облако (только localhost)
- Менять lib/ модули — только обёртка

## Приоритет
1. Лобби + выбор кампании
2. Игровой чат с персистентной историей
3. Создание кампании (визард)
4. Управление кампанией (дашборд, wiki)
